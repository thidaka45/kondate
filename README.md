# きょうのごはん（献立提案アプリ）

朝・昼・夕の献立をGemini APIが提案するPWAです。「かんたん家計簿」と同じ構成（HTML/CSS/JS + GitHub Pages）で作られています。

## ファイル構成

```
index.html   画面構造
style.css    デザイン（グリーン基調、ダークモード対応）
app.js       画面ロジック + Gemini API呼び出し
manifest.json / sw.js   PWA化（ホーム画面追加・簡易オフライン対応）
icons/       アプリアイコン
```

## 動かし方（ローカル確認）

1. このフォルダをそのままブラウザで開くか、`python3 -m http.server` などの簡易サーバーで配信
2. 右上の⚙アイコンからGemini APIキーを入力（初回は自動的に設定画面が開きます）
3. 朝/昼/夕を選び、条件タグと手持ちの食材を入力して「献立を提案してもらう」をタップ

## GitHub Pagesへのデプロイ

かんたん家計簿と同じ手順です。

```
thidaka45/kondate というリポジトリを作成
このフォルダの中身一式をpush
Settings → Pages → Branch: main / (root) を指定
```

公開後は `thidaka45.github.io/kondate` でアクセスでき、iPhoneでは「ホーム画面に追加」でアプリのように使えます。

## 料理写真について

Geminiでの画像自動生成は精度・コストの兼ね合いで見送り、代わりに各カードとレシピ詳細に以下のリンクを付けています。API呼び出しなしで、実際の料理写真やレシピをブラウザの別タブで確認・共有できます。

- **🔍 画像で見る** → 料理名でGoogle画像検索
- **📖 クックパッドで見る** → 料理名でクックパッド検索

## APIキーについて

Gemini APIキーは端末のlocalStorageにのみ保存され、Google以外の外部には送信されません。キーは [Google AI Studio](https://aistudio.google.com/) から無料で取得できます。

## 今後の拡張候補

- 提案から不足食材を抽出して「買い物」タブに自動反映
- 「かんたん家計簿」との連携（使った食材の概算コストを転記）
- レシート連携（既存のOCR機能を流用し、買った食材を自動で「手持ち食材」へ）
